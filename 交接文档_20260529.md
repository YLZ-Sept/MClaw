# MClaw 交接文档 — 2026/05/29

## 一、改动概览

### 1. 组织架构置顶
**文件**: `frontend/src/views/internal/HRManagement.vue`
- HR 侧栏 Tab 顺序调整为：**组织架构 → 员工档案 → 考勤月报 → 绩效管理**
- 默认 Tab 改为 `departments`

### 2. 考勤月报：导出 + 编辑
**后端** | `backend/routes/hr-attendance.js`
- `PUT /reports/:id` — 编辑单条考勤记录（动态字段更新）
- `GET /export?month=YYYY-MM` — 导出当月考勤 Excel（带列宽预设）

**前端 API** | `frontend/src/api/hr.js`
- `attendanceApi.updateReport(id, data)`
- `attendanceApi.exportUrl(month)`

**前端页面** | `HRManagement.vue`
- 工具栏新增「导出」按钮（无数据时置灰）
- 操作列新增「编辑」按钮 → 弹出编辑弹窗（基本信息 + 考勤概况/异常统计/假勤统计 三组字段）

### 3. 绩效管理三模块重构（重点）
**数据库** | `backend/db.js`
- `performance_reports` 表新增 `category` 字段（值：`finance`/`tech`/`monthly`）

**后端** | `backend/routes/hr-performance.js` — 整体重写

| 接口 | 功能 |
|------|------|
| `GET /reports?month=&category=` | 按月份+类别查询 |
| `POST /batch` | 批量入库（含 category） |
| `PUT /reports/:id` | 编辑单条（dims JSON 自动序列化） |
| `DELETE /reports/:id` | 删单条 |
| `POST /import` | 文件导入预览（按 category 用不同模板解析） |
| `GET /export?month=&category=` | 按类别导出 Excel |
| `POST /aggregate` | 月度考核汇总：合并财务+技术中心数据 |

- **财务模板**：4 维度（工作任务及质量 40%、工作态度 20%、协调能力 20%、学习能力 20%）
- **技术中心模板**：5 维度（工作效率 20%、工作质量 30%、执行反馈力 20%、专业技能 10%、团队协作 10%）
- **汇总逻辑**：同人两表都有 → 取平均分；仅一个表 → 直接用该表分
- **月度考核排序**：按总分 `DESC` 降序

**前端 API** | `frontend/src/api/hr.js`
```js
performanceApi = {
  reports(params),              // 加 category 参数
  importPreview(formData),      // FormData 内含 category
  batchInsert({month, category, records}),
  aggregate(month),             // 新增：触发月度汇总
  updateReport(id, data),       // 新增：编辑
  deleteReport(id),
  exportUrl(month, category)    // 加 category 参数
}
```

**前端页面** | `HRManagement.vue`
- 绩效 tab 内三子模块，`el-radio-group` 切换：**财务 | 商务/技术中心 | 月度考核**
- 财务和技术中心：导入 + 导出 + 编辑 + 删除
- 月度考核：汇总 + 导出 + 编辑 + 删除（无导入按钮）
- 月度考核表格简化：仅 序号/姓名/部门/职位/总分/操作（不显示维度列）
- 三表通用编辑弹窗：维度评分字段按行数据动态生成，保存时自动重算加权总分

## 四、关键设计说明

- 绩效考核三类数据存同一张 `performance_reports` 表，`category` 字段区分
- `dims` 列为 JSON 字符串，不同类别维度数不同，前端动态渲染列表头
- 月度考核汇总为一次性操作（先删当月旧数据再写新数据），可重复执行
- 编辑保存时会根据维度分值和权重重新计算 `total_score`
- 所有导出均为 xlsx 格式，服务器端生成后流式返回

### 推送流程

本机全局 git 配置了 `url.https://kkgithub.com/.insteadof=https://github.com/`，所有 GitHub 请求会被重写至 kkgithub 镜像。当 kkgithub 不可用时，按以下步骤绕过：

```bash
# 1. 临时移除全局重定向
git config --global --unset url.https://kkgithub.com/.insteadof
git config --global --unset credential.https://kkgithub.com.provider

# 2. 设置 token 认证并推送
git remote set-url origin https://<TOKEN>@github.com/YLZ-Sept/MClaw.git
git push origin main

# 3. 恢复全局重定向
git config --global url."https://kkgithub.com/".insteadof "https://github.com/"
git config --global credential."https://kkgithub.com".provider generic
```

### 注意事项

- GitHub token 需具备 `repo` 权限，在 https://github.com/settings/tokens 管理
- kkgithub.com 为 GitHub 镜像，正常时走镜像，不可用时回退直连
- 推送前确认 `.gitignore` 已排除 `.db` / `.db-shm` / `.db-wal` / `node_modules/` 等
