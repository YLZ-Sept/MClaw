# 蛙蔻面板 (MClaw V1.0) — 交接文档

> 更新日期：2026-05-30

---

## 1. 项目概述

企业智能体管理平台，基于 Vue3 + Element Plus + Express + SQLite，集成多模型 API（DeepSeek / MiMo / Anthropic / 智谱 / Kimi / 通义千问 / 豆包 / Ollama）。

核心功能：AI 聊天（SSE 流式）、CRM、进销存、人事、文档管理、FAQ、招投标。

---

## 2. 技术栈

| 层 | 技术 |
|----|------|
| 前端 | Vue 3 (Composition API) + Element Plus + Vue Router + Axios + Marked |
| 后端 | Express.js + better-sqlite3 |
| AI | OpenAI 兼容 API（当前默认小米 MiMo mimo-v2.5-pro） |
| 其他 | XLSX（Excel 导入导出）、mammoth（Word）、pdf-parse v2.x、multer（上传）、cheerio（网页抓取） |

---

## 3. 启动方式

```bash
# 后端
cd backend && node server.js          # 端口 :3001

# 前端
cd frontend && npx vite --host 0.0.0.0  # 端口 :5174
```

前端 Vite 代理 `/api` → `http://localhost:3001`，开发时无需额外配置。

## 5. 进销存模块（InventoryManagement.vue）

当前 5 个 Tab：

| Tab | 表名 | 后端路由 | 数据库 |
|------|------|----------|--------|
| 采购订单 | purchase_orders | `/api/purchase-orders` | 扁平表 |
| 销售订单 | sales_orders | `/api/sales-orders` | 扁平表 |
| 设备台账 | asset_ledger | `/api/asset-ledger` | 扁平表 |
| 退换货管理 | returns | `/api/returns` | 扁平表 |
| 库存预警 | asset_ledger（过滤） | 同上 | 同上 |

### asset_ledger 表结构（含本次新增 min_stock）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | UUID |
| manufacturer | TEXT | 厂商 |
| category | TEXT | 类别 |
| product_name | TEXT | 产品名称（必填） |
| model | TEXT | 型号 |
| unit | TEXT | 单位（默认"套"） |
| in_quantity | REAL | 入库数量 |
| out_quantity | REAL | 出库数量 |
| balance | REAL | 库存余量 |
| unit_price | REAL | 单价 |
| order_total | REAL | 合同订单总价 |
| inventory_value | REAL | 库存总价 |
| stock_date | TEXT | 入库时间 |
| min_stock | INTEGER | 预警阈值（默认 10）← 新增 |
| created_at | TEXT | 创建时间 |

### 导出端点

每个模块有独立导出，列头为中文，文件名为 `<模块>_数据.xlsx`：

```
GET /api/purchase-orders/export
GET /api/sales-orders/export
GET /api/asset-ledger/export
GET /api/returns/export
```

### 导入模板

通过 IO 系统下载：`GET /api/io/<key>/template`，配置在 `io-configs.js`。

## 7. 关键设计约定

- **ID 生成**：所有 ID 用 `crypto.randomUUID()`
- **DB 迁移**：通过 `try { ALTER TABLE ... ADD COLUMN } catch {}` 兼容旧表
- **导出**：后端用 `XLSX.utils.aoa_to_sheet()` 生成 Buffer，`window.open()` 触发下载
- **导入**：前端 ImportDialog 组件 → 后端 IO 系统（`/api/io/:key/import` 解析 + `/api/io/:key/batch` 批量写入）
- **API Key 脱敏**：列表返回 `***last4`，测试走 `POST /:id/test`
- **Vite 代理**：`/api` → `http://localhost:3001`

## 8. Agent 聊天流程

```
POST /api/chat/send {content, agent, stream, session_id}
  → 加载 Agent 配置 (systemPrompt + tools + 技能 + 知识库)
  → FAQ 预匹配（TF-IDF 向量检索）
  → 投诉关键词检测
  → 调用活跃模型 API（从 model_configs 动态读取）
  → 模型返回 tool_call → 执行工具 → 结果回传（最多 2 轮）
  → 流式输出文本 (SSE: tool/text/polished/done)
  → 润色轮 (二次 LLM 调用)
  → 存入历史
```

## 10. 常见问题

| 问题 | 处理 |
|------|------|
| 后端启动报端口占用 | `netstat -ano \| grep 3001` 找到 PID → `taskkill /PID xxx /F` |
| 前端热更新不生效 | Vite HMR 偶尔失效，硬刷新浏览器（Ctrl+Shift+R） |
| 导入 Excel 列不匹配 | 检查 `io-configs.js` 中该表的 columns 定义和 Excel 表头是否一致 |
| SQLite 列不存在报错 | 旧数据没有新列，确认 ALTER TABLE 已执行（try/catch 自动处理） |
| PDF 解析 | 使用 pdf-parse v2.x：`new PDFParse(new Uint8Array(buf))` → `load()` → `getText()` |
