# 蛙蔻面板 (MClaw V1.0) — 交接文档

> 更新日期：2026-05-30

---

## 1. 项目概述

企业智能体管理平台，基于 Vue3 + Element Plus + Express + SQLite，集成多模型 API（DeepSeek / MiMo / Anthropic / 智谱 / Kimi / 通义千问 / 豆包 / Ollama）。

核心功能：AI 聊天（SSE 流式）、CRM、进销存、人事、文档管理、FAQ、招投标。

---

## 2. 技术栈

| 层 | 技术 |
|----|------|
| 前端 | Vue 3 (Composition API) + Element Plus + Vue Router + Axios + Marked |
| 后端 | Express.js + better-sqlite3 |
| AI | OpenAI 兼容 API（当前默认小米 MiMo mimo-v2.5-pro） |
| 其他 | XLSX（Excel 导入导出）、mammoth（Word）、pdf-parse v2.x、multer（上传）、cheerio（网页抓取） |

---

## 3. 启动方式

```bash
# 方式一：双击一键启动
start.bat          # 自动杀旧进程 → 启后端(3001) → 等 4 秒 → 启前端(5174)

# 方式二：手动
cd backend  && node server.js          # → :3001
cd frontend && npx vite --host 0.0.0.0 # → :5174
```

前端 Vite 代理 `/api` → `http://localhost:3001`，开发时无需额外配置。

## 5. 进销存模块（InventoryManagement.vue）

当前 5 个 Tab：

| Tab | 表名 | 后端路由 | 数据库 |
|------|------|----------|--------|
| 采购订单 | purchase_orders | `/api/purchase-orders` | 扁平表 |
| 销售订单 | sales_orders | `/api/sales-orders` | 扁平表 |
| 设备台账 | asset_ledger | `/api/asset-ledger` | 扁平表 |

### asset_ledger 表结构（含本次新增 min_stock）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | UUID |

| unit_price | REAL | 单价 |
| order_total | REAL | 合同订单总价 |
| inventory_value | REAL | 库存总价 |
| stock_date | TEXT | 入库时间 |

### 导出端点


### 导入模板

通过 IO 系统下载：`GET /api/io/<key>/template`，配置在 `io-configs.js`。

## 6. 人事模块（HRManagement.vue）

当前 10 个 Tab，顺序：**组织架构 → 员工档案 → 部门管理 → 招聘管理 → 候选人库 → 假期管理 → 考勤月报 → 考勤规则 → 绩效管理 → 异动管理**。

### 6.1 考勤月报

**后端** `backend/routes/hr-attendance.js`：
- `PUT /reports/:id` — 编辑单条（动态字段更新）
- `GET /export?month=YYYY-MM` — 导出当月考勤 Excel（带列宽预设）

**前端**：工具栏「导出」按钮（无数据时置灰），操作列「编辑」按钮 → 弹窗编辑基本信息 + 考勤概况/异常统计/假勤统计三组字段。

### 6.2 绩效管理（重点）

三类数据存同一张 `performance_reports` 表，`category` 字段区分（`finance` / `tech` / `monthly`）。

前端 `el-radio-group` 切换三个子模块：

| 子模块 | category | 功能 |
|--------|----------|------|
| 财务 | `finance` | 导入 + 导出 + 编辑 + 删除 |
| 商务/技术中心 | `tech` | 导入 + 导出 + 编辑 + 删除 |
| 月度考核 | `monthly` | 汇总 + 导出 + 编辑 + 删除（无导入） |

**维度模板**：
- **财务**：4 维度 — 工作任务及质量 40%、工作态度 20%、协调能力 20%、学习能力 20%
- **技术中心**：5 维度 — 工作效率 20%、工作质量 30%、执行反馈力 20%、专业技能 10%、团队协作 10%
- `dims` 列为 JSON 字符串，不同类别维度数不同，前端动态渲染列表头

**后端** `backend/routes/hr-performance.js`：

| 接口 | 功能 |
|------|------|
| `GET /reports?month=&category=` | 按月份+类别查询 |
| `POST /batch` | 批量入库（含 category） |
| `PUT /reports/:id` | 编辑单条（dims JSON 自动序列化，自动重算 weighted total） |
| `DELETE /reports/:id` | 删单条 |
| `POST /import` | 文件导入预览（按 category 用不同模板解析） |
| `GET /export?month=&category=` | 按类别导出 Excel |
| `POST /aggregate` | 月度考核汇总：合并财务+技术中心数据 |

**汇总逻辑**：同人两表都有 → 取平均分；仅一个表 → 直接用该表分。先删当月旧数据再写。按总分 DESC 降序排列。

**模板文件路径**（本地）：
```
G:\桌面\人力资源管理模块\绩效考核\
  ├── 财务、商务.xlsx    ← 导入财务子模块
  ├── 技术中心.xlsx      ← 导入商务/技术中心子模块
  └── 月度考核.xlsx      ← 导出模板（汇总后导出）
```

## 8. 关键设计约定

- **ID 生成**：所有 ID 用 `crypto.randomUUID()`
- **DB 迁移**：通过 `try { ALTER TABLE ... ADD COLUMN } catch {}` 兼容旧表
- **导出**：后端用 `XLSX.utils.aoa_to_sheet()` 生成 Buffer，`window.open()` 触发下载
- **导入**：前端 ImportDialog 组件 → 后端 IO 系统（`/api/io/:key/import` 解析 + `/api/io/:key/batch` 批量写入）
- **API Key 脱敏**：列表返回 `***last4`，测试走 `POST /:id/test`
- **Vite 代理**：`/api` → `http://localhost:3001`
- **PDF 解析**：pdf-parse v2.x — `new PDFParse(new Uint8Array(buf))` → `load()` → `getText()`

## 9. Agent 聊天流程

```
POST /api/chat/send {content, agent, stream, session_id}
  → 加载 Agent 配置 (systemPrompt + tools + 技能 + 知识库)
  → FAQ 预匹配（TF-IDF 向量检索）
  → 投诉关键词检测
  → 调用活跃模型 API（从 model_configs 动态读取）
  → 模型返回 tool_call → 执行工具 → 结果回传（最多 2 轮）
  → 流式输出文本 (SSE: tool/text/polished/done)
  → 润色轮 (二次 LLM 调用)
  → 存入历史
```

## 10. Git 推送

| 项 | 值 |
|----|----|
| 远程仓库 | `https://github.com/YLZ-Sept/MClaw.git` |
| 分支 | `main` |

本机全局 git 配置了 `url.https://kkgithub.com/.insteadof=https://github.com/`，所有 GitHub 请求重写至 kkgithub 镜像。当 kkgithub 不可用时按以下步骤绕过：

```bash
# 1. 临时移除全局重定向
git config --global --unset url.https://kkgithub.com/.insteadof
git config --global --unset credential.https://kkgithub.com.provider

# 2. 设置 token 认证并推送
git remote set-url origin https://<TOKEN>@github.com/YLZ-Sept/MClaw.git
git push origin main

# 3. 恢复全局重定向
git config --global url."https://kkgithub.com/".insteadof "https://github.com/"
git config --global credential."https://kkgithub.com".provider generic
```

注意事项：
- GitHub token 需具备 `repo` 权限
- `.gitignore` 已排除 `.db` / `.db-shm` / `.db-wal` / `node_modules/`
- 推送前确认无敏感文件混入

## 12. 常见问题

| 问题 | 处理 |
|------|------|
| 后端启动报端口占用 | `netstat -ano \| grep 3001` 找到 PID → `taskkill /PID xxx /F` |
| 前端热更新不生效 | Vite HMR 偶尔失效，硬刷新浏览器（Ctrl+Shift+R） |
| 导入 Excel 列不匹配 | 检查 `io-configs.js` 中该表的 columns 定义和 Excel 表头是否一致 |
| SQLite 列不存在报错 | 旧数据没有新列，确认 ALTER TABLE 已执行（try/catch 自动处理） |
